# Backend Implementation — Overview

Purpose

Este diretório contém a documentação de implementação do backend do Shippa One. Ele traduz as decisões arquiteturais e os requisitos em documentação executável para engenheiros backend e DevOps.

Files in this folder

- auth.md — autenticação, sessões, refresh tokens e middleware
- database.md — modelo de dados, padrões de tabelas e RLS
- api.md — convenções de API e contratos importantes
- migrations.md — processo de migrações e runbook
- security.md — controles de segurança obrigatórios e checklist
- sessions.md — desenho de sessões e cookies HttpOnly
- roles_permissions.md — modelo de papéis e permissões
- observability.md — logs, métricas, tracing e health checks
- ci_cd.md — pipeline, checks obrigatórios e políticas de merge
- deployment.md — ambientes e runbook de deploy

Status

Draft — atualize conforme aprovações e decisões finais de arquitetura.
