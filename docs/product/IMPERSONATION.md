# Impersonation (Acessar como Cliente)

Objetivo

Permitir que administradores autorizados acessem temporariamente a conta de um cliente para fins de suporte.

Regras

- Disponível apenas para perfis autorizados.
- Motivo do acesso obrigatório.
- Banner visível durante toda a sessão.
- Todas as ações registradas em audit_logs.
- Sessão temporária.
- Encerramento manual ou automático por inatividade.
- Cliente pode visualizar o histórico de acessos.
- Integração futura com chamados de suporte para registrar a autorização do cliente.
